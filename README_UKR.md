# Numismatist — Облік монет

[![WordPress](https://img.shields.io/badge/WordPress-≥6.0-blue)](https://wordpress.org/)
[![PHP](https://img.shields.io/badge/PHP-≥8.0-purple)](https://php.net/)
[![License](https://img.shields.io/badge/License-GPLv2-green)](LICENSE)
[![Version](https://img.shields.io/badge/Version-1.3.0-orange)](CHANGELOG)

Мультикористувацький AJAX-плагін для WordPress, що дозволяє вести облік нумізматичних колекцій. Кожен зареєстрований користувач має власну ізольовану колекцію монет. Таблиця розміщується на будь-якій сторінці сайту за допомогою шорткоду `[numismatist]`.

## Швидкий старт

1. Скопіюйте папку `Numismatist/` у `wp-content/plugins/`.
2. Активуйте плагін у розділі **Плагіни → Встановлені плагіни**.
3. Створіть нову сторінку, вставте шорткод `[numismatist]`, опублікуйте.
4. Відкрийте сторінку як зареєстрований користувач — колекція готова до роботи.

---

## Шорткод

```
[numismatist]
```

| Тип відвідувача | Що бачить |
|---|---|
| **Авторизований користувач** | Свою колекцію монет + кнопка «ДОДАТИ» + іконки редагування ✏ та видалення 🗑 у кожному рядку + модальне вікно з формою |
| **Гість (не авторизований)** | Порожню таблицю у режимі перегляду |

Шорткод можна розмістити на кількох сторінках одночасно.

## Можливості

- **Мультикористувацька ізоляція** — поле `id_user` (= `wp_users.ID`) у таблиці монет; кожен SQL-запит містить `WHERE id_user = %d`. Доступ до чужих записів неможливий на рівні бази даних.
- **AJAX-інтерфейс** — пошук, фільтрація, пагінація та всі CRUD-операції без перезавантаження сторінки.
- **Будь-який авторизований користувач** — право `manage_options` не потрібне; достатньо бути зареєстрованим на сайті.
- **Автоматичне оновлення схеми БД** — міграції запускаються через хук `plugins_loaded` при виявленні нової версії. Файли можна оновлювати через rsync/FTP без повторної активації плагіну.
- **Медіатека WordPress** — стандартний інструмент вибору/завантаження фото монет.
- **Компактні іконки дій** — SVG-іконки олівця та кошика замість текстових кнопок у рядках таблиці.
- **Жовта кнопка «Скасувати»** — візуальне розмежування деструктивних та нейтральних дій у формі.
- **HTTP 200 для всіх AJAX-відповідей** — `wp_send_json_error()` не встановлює коди 4xx/5xx, тому jQuery `.fail()` не перехоплює відповіді та реальні повідомлення про помилки завжди доходять до користувача.
- **Єдиний CSS-файл** — стилі адмін-панелі та фронтенду зосереджені в `css/numismatist.css`. Підключення лише на сторінках де потрібно.

## Встановлення та оновлення

### Нова установка

```bash
# Скопіювати папку плагіну
cp -r Numismatist/ /var/www/html/wp-content/plugins/

# Активувати через WP CLI (або через адмін-панель)
wp plugin activate Numismatist
```

### Оновлення з попередньої версії

```bash
# Розгорнути нові файли (деактивація НЕ потрібна)
rsync -avz --delete Numismatist/ user@server:/path/to/plugins/Numismatist/
```

Хук `plugins_loaded` автоматично виявить зміну версії та запустить міграції при наступному запиті до сайту.

> **Оновлення з v1.0–v1.2.0:** колонка `id_user` буде автоматично додана або перейменована з `user_id` без втрати даних.

## Структура бази даних

Таблиця: `{prefix}coins`

| Колонка | Тип | Опис |
|---|---|---|
| id | BIGINT UNSIGNED | Первинний ключ, автоінкремент |
| **id_user** | **BIGINT UNSIGNED** | **ID користувача WordPress (wp_users.ID) — ізоляція даних** |
| name | VARCHAR(255) | Назва монети (обов'язкове) |
| url | VARCHAR(2048) | Зовнішнє посилання |
| year | SMALLINT | Рік випуску |
| material | VARCHAR(100) | Матеріал (наприклад: Срібло, Мідь) |
| circulation | VARCHAR(100) | Тираж |
| price | DECIMAL(10,2) | Орієнтовна ціна |
| photo | VARCHAR(2048) | URL фото з медіатеки WordPress |
| quantity | INT UNSIGNED | Кількість примірників |
| notes | TEXT | Довільні нотатки |
| sorting | INT | Ручне сортування (за замовчуванням 0) |
| created_at | DATETIME | Дата/час створення (встановлюється автоматично) |
| updated_at | DATETIME | Дата/час оновлення (оновлюється автоматично) |

**Індекси:** `idx_id_user`, `idx_year`, `idx_material`, `idx_sorting`

## Розгортання

### На VPS (rsync)

```bash
# Налаштуйте змінні у файлі та запустіть:
bash deploy-vps.sh
```

Скрипт виконує:
1. Перевіряє наявність вихідної директорії.
2. Синхронізує файли через rsync (з виключенням `.git`, `.idea`, `.DS_Store`).
3. Встановлює власника `www-data` та права `755/644` через SSH.

### У локальне Docker-середовище

```bash
bash deploy-local.sh
```

## Безпека

| Механізм | Реалізація |
|---|---|
| CSRF-захист | Перевірка nonce (`wp_verify_nonce`) у кожному AJAX-обробнику |
| Аутентифікація | `is_user_logged_in()` — лише авторизовані користувачі |
| Ізоляція даних | `WHERE id_user = %d` у кожному SQL-запиті |
| SQL-ін'єкції | `$wpdb->prepare()` для всіх запитів; `$wpdb->insert/update` з форматами |
| XSS на виводі | `esc_html()`, `esc_attr()`, `esc_url()`, `esc_js()` + клієнтська функція `escHtml()` |
| Санітизація вводу | `sanitize_text_field()`, `absint()`, `esc_url_raw()`, `sanitize_textarea_field()` |
| Захист файлів | `if (!defined('ABSPATH')) exit;` у кожному PHP-файлі |
| AJAX HTTP-статус | Завжди HTTP 200 — помилки через `success:false` у JSON |
